\begin{abstract}
Single sign-on (SSO) allows a user to maintain only the credential for an identity provider (IdP) to log into multiple relying parties (RPs).
However, SSO introduces privacy threats, as ({\em a}) a curious IdP could track a user's all visits to RPs, and ({\em b}) colluding RPs could learn a user's online profile by linking her identities across these RPs.
This paper presents a privacy-preserving SSO scheme, called \usso, to protect a user's online profile against both (\emph{a}) an honest-but-curious IdP and (\emph{b}) malicious RPs colluding with other users.
An \emph{identity-transformation} approach is proposed to generate \emph{untraceable} \emph{ephemeral pseudo-identities} for an RP and a user in a login, %(denoted as $PID_{RP}$ and $PID_U$, respectively)
 from which the visited RP derives a \emph{permanent account} for the user, %(denoted as $Acct$),
  while the transformations provide \emph{unlinkability}.
This approach protects the identities of a user and the visited RP,
 while % working compatibly with widely-deployed SSO protocols and
 enabling SSO services accessed from a commercial-off-the-shelf browser without plug-ins or extensions. %and satisfying security requirements.
We built a prototype of \usso\ on top of MITREid Connect, an open-source SSO system. The extensive evaluations show that it fulfills the security and privacy requirements of SSO
with reasonable overheads.
%Compared with existing SSO protocols, \usso\ is among the first to prevent both types of privacy threats. Compared with existing privacy-preserving identity federation schemes, \usso\ supports all the desired SSO features while providing sufficient privacy protections.  % 省去，放到Introduction即可

In each SSO login, an \emph{ephemeral pseudo-identity} of an RP, denoted as $PID_{RP}$, is negotiated between a user and the RP. $PID_{RP}$ is sent to the IdP and designated in the identity token, so that the IdP is not aware of the visited RP.

\end{abstract}
